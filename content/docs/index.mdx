## Start with the resource hierarchy

Most day-to-day work follows this hierarchy:

1. An **organization** owns projects, teams, users, billing, and usage.
2. A **team** groups users and permissions within an organization.
3. A **project** contains deployable resources and project-scoped configuration.
4. A **resource** is a workload such as a container service, database, or game server.

Continue to [Platform concepts](/docs/platform) for the ownership model, or [Products](/docs/products) for workload capabilities.

## Documentation scope

These docs are derived from the current Fabort codebase. Features that are administrator-only are identified separately from frontend workflows intended for organization users.
